# LEXON Derivation Trace Format

LEXON derivation traces record why an obligation was classified as `Applicable`, `NotApplicable`, or `Uncertain`, and why evidence gaps or conflicts were reported.

## Trace fields

Each trace includes:

- `trace_id` — unique identifier
- `trace_type` — `"activation"` or `"conflict"`
- `clause_id` — source clause identifier
- `obligation_id` — obligation identifier within the clause
- `system_id` — AI system profile identifier
- `activation_status` — `"Applicable"`, `"NotApplicable"`, or `"Uncertain"`
- `actor_match` — boolean
- `conditions` — list of condition evaluations (predicate, expected, observed, truth_value)
- `exceptions_fired` — list of exception identifiers that fired
- `required_evidence` — list of required evidence identifiers
- `held_evidence` — list of evidence identifiers held by the system
- `evidence_gaps` — list of required evidence items not held
- `rules_fired` — list of rule identifiers that contributed to the derivation
- `human_review_required` — boolean
- `notes` — free-text notes, always including the non-advice disclaimer

## Trace files

Generated by `scripts/generate_trace_examples.py`:

```
outputs/traces/trace_applicable_CL-002_SYS-007.json
outputs/traces/trace_not_applicable_CL-002_SYS-013.json
outputs/traces/trace_uncertain_CL-010_SYS-027.json
outputs/traces/trace_conflict_example.json
```

## Purpose

Trace files are intended for auditability and reproducibility. They are not legal opinions.
